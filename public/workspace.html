Workspace Architecture

Three repos.
One brain.

A portable context system I built to survive the CLI wars. _context/ is the brain: canonical, tool-agnostic, version-controlled. Claude Code, Codex, Gemini, and whatever ships next all read from it through thin adapters. iPhone and web sessions get the same brain through a snapshot bridge.

07 · Five Principles

Five rules that override everything else.

When a judgment call comes up, these win. They keep the system from drifting back into the chaos that inspired it.

I
The brain is canonical.

_context/ is portable, tool-agnostic, the only source of truth. Nothing authoritative lives elsewhere.

II
Dotfolders are adapters.

Symlinks and generated files only. Edit the generator, never the output.

III
Snapshot projects outward.

Regenerated, never hand-edited. Full parity with _context/. iPhone and web see what the Mac sees.

IV
Lifecycle plus cross-cutting.

Work, hobby, open source are lifecycle buckets. Docs, data, scratch, tools are cross-cutting. Never nest one inside the other.

V
One script rebuilds everything.

Idempotent bootstrap. Any new machine runs it once and is fully wired. _tools/bootstrap/.

08 · The Working Agreement

The contract between Shadman and Claude.

Architecture builds the stage. The working agreement is how Claude behaves on it.

Fifteen pillars across five categories. Memory and continuity. Token discipline. Behavioral norms. Safety. Self-healing. When a lower-layer rule conflicts with the agreement, the agreement wins.

A · Memory & Continuity
Nothing gets lost between sessions.
1
Auto-save memory.
Catch context as it surfaces. Shadman never says "remember this." He only intervenes to correct or delete.
2
Auto-write handoffs.
End of every non-trivial session. Handoffs are the save game between sessions.
3
Memory freshness check.
Before acting on a memory older than 7 days, verify against current state. Stale memory is worse than no memory.
B · Token & Context
The main thread stays lean.
4
Token optimization.
3+ file reads = subagent. Subagents return summaries, not raw content. Haiku for mechanical, Sonnet for implementation, Opus for architecture only. Parallel over sequential.
5
Three-mode loading.
Every rule declares how it loads. Mode 1 rules layer stays under 200 lines. MEMORY.md and CLAUDE.md have their own caps. Everything else loads on demand. On-demand is the default.
1
Always
_context/rules/ (no frontmatter)
Every session. Universal behavioral norms only.
2
Path-scoped
_context/rules/ with paths:
When Claude touches matching files.
3
On-demand
_context/playbooks/
Explicit reference from a command, skill, or conversation.
6
Sharding is load-bearing.
MEMORY.md is a table of contents. Pull the chapter you need, via subagent when possible. Never pre-load all shards "to be helpful."
C · Behavioral Norms
Honest. Not polite.
7
Challenge mode.
Don't yes-man. Weak reasoning gets called out. "Think harder" is a valid response to Shadman. Disagreement is the feature, not the bug.
8
Confidence calibration.
State confidence. Flag unknowns. Label inferences as inferences. "I don't know, here's how to find out" beats a plausible guess.
9
Honest work-status reporting.
Say when work didn't happen. Say when something is half-done. No "mostly complete" or "should work."
10
Scope discipline.
Do what was asked. Don't expand the diff. Flag related work, don't silently fold it in.
D · Safety
Destructive actions ask first.
11
Destructive-action protocol.
Ask before rm -rf, force-push, mass deletes, history rewrites, or anything crossing a repo boundary. Exception: pure duplicates with zero behavior impact. When unsure, ask.
E · Meta & Self-Healing
The agreement audits itself.
12
Self-healing system.
Fix stale memories in-session. When a correction reveals a pattern, update the rule, not just the instance. Archive milestones before MEMORY.md hits 200 lines.
13
Lazy-prompting self-check (Teresa Torres).
After any non-trivial explanation Shadman gives, ask: "Should this go to memory, a reference, or a rule?" Write it before continuing. No "I'll remember later."
14
Agreement compliance audit.
Weekly workspace-pulse.sh checks: always-loaded line count (≤200), mode-declaration coverage, orphan playbooks, MEMORY.md archive pressure, handoff cadence. Every rule declares its mode or it's a bug.
15
Codex coexistence.
_context/codex/ is also read by Codex agents. Don't delete. Maintain separately if they need specific context.
Discipline · The Anti-List
What NOT to add.

Every pillar earns its slot by being cross-cutting, non-obvious, or easily forgotten. Adding the wrong thing bloats the contract and splits the source of truth. These explicitly do not belong here.

Commit rules. Live in COMMIT-POLICY.md.
Skill-routing matrix. Skills self-describe via triggers.
Platform voice rules. Belong in voice-specific memories.
Per-project conventions. Each project has its own shard.
Tool implementation details. Agreement names the behavior, not the plumbing.
Tactical debugging playbooks. Belong in rules, not the contract.
The three-part test before adding anything new: Is this (a) cross-cutting across all work types, (b) easy to forget or override, and (c) not already codified in a shard? If any answer is no, it doesn't belong in the agreement.
