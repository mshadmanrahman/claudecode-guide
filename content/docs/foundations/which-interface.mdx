## Use the terminal. Unless you're not a developer, in which case use the web app.

That's the honest answer. Everything else is nuance around it.

Most guides present this as "it depends on your use case." That's true, but not very useful. So: here's what I actually recommend based on who you are and where you're starting from.

## The Quick Decision

| You are... | Use this | Why |
|---|---|---|
| A developer comfortable in the terminal | **Terminal (<VocabBridge term="CLI" explanation="Command Line Interface: a way to use software by typing text commands instead of clicking buttons.">CLI</VocabBridge>)** | Maximum power, full extensibility |
| A developer who lives in VS Code | **VS Code <VocabBridge term="Extension" explanation="An add-on that gives extra features to a program, like installing a new app on your phone.">Extension</VocabBridge>** | Same power, never leave your editor |
| A developer who lives in JetBrains | **JetBrains Extension** | Same power, native <VocabBridge term="IDE" explanation="Integrated Development Environment: a fancy text editor designed for writing code, with built-in tools for testing and debugging.">IDE</VocabBridge> integration |
| A PM or non-developer | **Desktop App** or **Web App** | No terminal knowledge needed |
| Trying Claude Code for the first time | **Web App** | Zero install, instant start |
| On a shared/company machine with restrictions | **Web App** | Nothing to install |

## Terminal (CLI): The Power User's Choice

This is where Claude Code started and where it's still most powerful. If you're comfortable in the terminal, this is probably where you should be.

**What you get:**
- Full access to <VocabBridge term="hooks" explanation="Automatic triggers that run actions when something happens, like 'when I save a file, check it for errors'.">hooks</VocabBridge>, skills, <VocabBridge term="MCP servers" explanation="Plugins that connect Claude to tools like GitHub, Slack, or databases so it can work with them directly.">MCP servers</VocabBridge>, and <VocabBridge term="sub-agents" explanation="Independent AI helpers that work on separate tasks in parallel, like having multiple assistants.">sub-agents</VocabBridge>
- Direct <VocabBridge term="filesystem" explanation="Your computer's file and folder system: everything stored on your hard drive.">filesystem</VocabBridge> access (Claude Code reads and writes your actual files)
- Works alongside any editor: VS Code, Vim, Zed, Emacs, whatever you prefer
- Configuration via `CLAUDE.md` files that live in your repo and version-control your AI behavior
- Autonomous loops and background tasks

**What you give up:**
- You need to be comfortable typing commands in a terminal
- No visual diff preview before edits happen (you see diffs after)
- Steeper initial learning curve than the GUI surfaces

## IDE Extensions: Best of Both Worlds

The VS Code and JetBrains extensions embed Claude Code directly inside your editor. Same agent, same capabilities, but you never switch windows.

**What you get:**
- Claude Code in a panel inside your editor
- Same filesystem access, same hooks, same skills as the CLI
- Easier to track which files are changing as work happens
- No context switching between terminal and editor

**What you give up:**
- The panel takes up screen real estate
- Extensions sometimes lag a version or two behind the CLI on new features

## Web App (claude.ai/code): Zero Friction

The web app gives you Claude Code in a browser tab. No installation, no terminal, no PATH configuration. Open the URL and start working.

**What you get:**
- Instant start: nothing to install or configure
- Works from any computer, including Chromebooks and locked-down work machines
- Same core agentic capabilities for code generation and analysis

**What you give up:**
- No local filesystem access (works with remote repos or uploaded code)
- Hooks, MCP servers, and advanced config aren't available
- Less integrated with your local development environment overall

**Best for:** First-time users, PMs exploring what Claude Code can do, quick tasks when you're away from your dev machine, evaluating whether it's worth installing at all.

## Desktop App: The Native Experience

The Claude Code desktop app (Mac and Windows) is the terminal CLI wrapped in a polished native application. Same capabilities, different container.

**What you get:**
- Native app experience: Cmd+Tab to switch, system notifications, dedicated window
- Full agent capabilities, same as the CLI
- Cleaner interface than raw terminal

**What you give up:**
- Functionally it's the same as the CLI. The differences are mostly aesthetic.

**Best for:** People who prefer apps over terminals, or anyone who wants Claude Code as a "first-class" application rather than a terminal command they type.

## The Real Answer: Start Anywhere, Graduate to Terminal

If you've never used Claude Code, start with the web app. Zero friction. You'll know within 10 minutes if this tool fits how you work.

If you like it and want more power, install the CLI or the IDE extension depending on where you spend your day. VS Code users should grab the extension. Terminal people should use the CLI. Both are the right answer for different people.

Frankly speaking, if you become a power user building skills, running hooks, using MCP servers, setting up autonomous loops, you'll end up in the terminal regardless. That's where the full configuration system lives. The other surfaces are on-ramps to the same destination, not competitors to it.

Pick the one that matches where you are today. You can move up anytime.

## Quick Setup per Surface

| Surface | How to start |
|---|---|
| **Terminal** | `npm install -g @anthropic-ai/claude-code && claude` |
| **VS Code** | Install "Claude Code" from VS Code marketplace |
| **JetBrains** | Install "Claude Code" from JetBrains plugin marketplace |
| **Web App** | Go to [claude.ai/code](https://claude.ai/code) |
| **Desktop** | Download from [claude.ai](https://claude.ai) under the Claude Code section |
