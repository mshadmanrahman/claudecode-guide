## Plans at a glance

| | Claude Pro | Claude Max 5x | Claude Max 20x | API (Pay-per-token) |
|---|---|---|---|---|
| **Cost** | $20/month | $100/month | $200/month | Opus: $5/$25, Sonnet: $3/$15 per 1M tokens |
| **Usage level** | Moderate | Heavy | Very heavy | Pay for what you use |
| **Rate limits** | Standard | 5x Pro limits | 20x Pro limits | Based on tier/spend |
| **Best for** | Getting started, light-to-moderate daily use | Full-time coding with Claude Code | All-day heavy agentic workflows | Building apps, CI/CD pipelines, programmatic use |
| **Access to** | Sonnet + Opus | Sonnet + Opus | Sonnet + Opus | All models via API |

## When Pro is enough

Pro covers most people. If you're using Claude Code for a few focused tasks per day — write a feature in the morning, debug something after lunch, refactor a module before end of day — you're probably fine. The rate limits exist but they're not aggressive for normal usage.

Signs Pro is working for you: you rarely see "rate limit" messages, you use Claude Code for 1-3 focused sessions per day, and you're still figuring out your workflow. Don't pay more until you've hit the ceiling.

## When to upgrade to Max

Long story short: you'll feel it. The signal is rate limits hitting during your actual working hours, not just when you're abusing it. If you're watching the clock waiting for limits to reset, or timing your prompts to avoid hitting the ceiling, the $80/month jump from Pro to Max 5x pays for itself fast.

Signs you need Max: you hit rate limits multiple times per week, you run long agentic sessions (30+ minutes of continuous back-and-forth), you use sub-agents or parallel workflows that consume tokens quickly, or your work is sitting idle waiting for limits to clear.

**5x or 20x?** Start with 5x ($100/month). Most heavy users find it sufficient. 20x is for people running Claude Code continuously with complex multi-agent workflows, autonomous loops, or very large codebases that chew through context. I'm guessing fewer than 10% of users actually need it.

## When the API makes sense

The API isn't really a "plan" for personal coding. It's for building things.

If you're integrating Claude into your own application, running it in CI/CD pipelines, or building tools that call Claude programmatically — that's API territory. The pricing model (per token, per request) makes sense when you need fine-grained cost control or automated access.

Don't use the API just because it sounds more technical. For personal coding sessions, Pro or Max is almost always simpler and cheaper.

## The actual cost math

A typical coding session with Claude Code uses somewhere between 50,000 and 200,000 tokens, depending on how large your codebase is and how many files it reads. At Sonnet API pricing, that's roughly $0.15 to $3.00 per session. Three to five sessions a day adds up to $1.35 to $15.00 per day. Pro at $20/month is cheaper than a single heavy week of API usage.

The API gets more economical for targeted, low-token tasks at high volume, or when you need programmatic access the CLI doesn't offer.

## Recommendation

Start with Pro. It's $20/month and covers most people while they learn what their actual usage looks like. Upgrade to Max 5x when you find yourself waiting for rate limits, not before. The API is for building products. Don't overthink it — you can switch plans anytime.
