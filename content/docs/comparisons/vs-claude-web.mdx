## "I Already Use Claude. What's Different?"

This is the #1 question we get. You go to claude.ai, you chat with Claude, it's great. Then someone mentions "Claude Code" and you think it's the same thing with a different name.

It's not. They're as different as Google Docs and Google Cloud.

**Claude.ai** is a chatbot. You type, it responds. Text in, text out.

**Claude Code** is an agent. It reads your files, writes code, runs commands, uses tools, and builds things on your computer.

Same AI brain. Completely different body.

## The Quick Comparison

| | Claude.ai (Chat) | Claude Code (Agent) |
|---|---|---|
| **What you do** | Type messages, get text back | Describe goals, watch them happen |
| **File access** | Upload files one at a time | Reads your entire project folder |
| **Can it edit files?** | No (gives you text to copy) | Yes, directly |
| **Can it run commands?** | No | Yes (terminal commands, tests, deploys) |
| **Can it use tools?** | Limited (artifacts, analysis) | Full (GitHub, Slack, Jira, databases via MCP) |
| **Memory** | Per-project memory (limited) | Rich memory system (preferences, projects, feedback) |
| **Custom automation** | No | Skills, hooks, sub-agents |
| **Computer Use** | No | Yes (sees your screen, clicks, navigates) |
| **Where it runs** | Browser tab | Your computer + optionally browser |

## When to Use Claude.ai

- **Quick questions.** "Explain this error message" or "what does this code do?"
- **Document writing.** Drafting emails, reports, blog posts where you don't need file access
- **Analysis.** Upload a CSV, ask questions about it
- **Mobile.** Claude.ai works great on your phone
- **Brainstorming.** Back-and-forth ideation where you're exploring, not building

## When to Use Claude Code

- **Building anything.** Websites, apps, scripts, automations
- **Working in a project.** Claude Code sees your whole codebase, not just one file
- **Multi-step tasks.** "Fix this bug, add a test, update the docs, commit"
- **Automation.** Skills and hooks that run on triggers
- **Tool integration.** Connect to GitHub, Slack, Jira, calendars, databases
- **Debugging.** Read logs, trace errors, check git history, propose fixes
- **Deployment.** Push to Vercel, Railway, or any platform

Most power users keep both open:
