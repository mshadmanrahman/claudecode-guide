## "I Already Use Claude. What's Different?"

This is probably the most common question I get. You go to claude.ai, you chat with Claude, it's smart and helpful. Then someone mentions "Claude Code" and you assume it's just a rebranded version of the same thing.

It's not. They're as different as Google Docs and Google Cloud. Same company, completely different products.

**Claude.ai** is a chatbot. You type, it responds. Text in, text out.

**Claude Code** is an agent. It reads your files, writes code, runs commands, uses tools, and builds things on your actual computer.

Same AI brain. Completely different body.

## The Quick Comparison

| | Claude.ai (Chat) | Claude Code (Agent) |
|---|---|---|
| **What you do** | Type messages, get text back | Describe goals, watch them happen |
| **File access** | Upload files one at a time | Reads your entire project folder |
| **Can it edit files?** | No (gives you text to copy) | Yes, directly |
| **Can it run commands?** | No | Yes (terminal commands, tests, deploys) |
| **Can it use tools?** | Limited (artifacts, analysis) | Full (GitHub, Slack, Jira, databases via MCP) |
| **Memory** | Per-project memory (limited) | Rich memory system (preferences, projects, feedback) |
| **Custom automation** | No | Skills, hooks, sub-agents |
| **Computer Use** | No | Yes (sees your screen, clicks, navigates) |
| **Where it runs** | Browser tab | Your computer + optionally browser |

## When to Use Claude.ai

Claude.ai is the right choice for several things.

Quick questions where you need a fast answer and don't want to open a terminal. Document writing, emails, reports, blog drafts where file access isn't the point. Analysis on an uploaded CSV or PDF. Anything on mobile, since Claude.ai works great on your phone. Back-and-forth ideation where you're thinking out loud, not building anything yet.

## When to Use Claude Code

Claude Code is the right choice when you're actually making something.

Building websites, apps, scripts, automations. Working inside an existing project where context matters. Multi-step tasks like "fix this bug, add a test, update the docs, commit." Setting up automation that runs on triggers. Connecting to GitHub, Slack, Jira, databases. Debugging something that requires reading logs and checking git history. Deploying to Vercel or any other platform.

Long story short: if you're creating or changing files, use Claude Code.

## The Best of Both Worlds

Most power users keep both running at the same time.

Claude.ai in a browser tab for quick questions and writing. Claude Code in the terminal or desktop app for building and automation. They complement each other rather than replacing each other. One is your thinking partner, the other is your building partner.
