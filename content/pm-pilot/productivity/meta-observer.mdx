## What it does

/meta-observer runs alongside your session and logs behavioral patterns: which skills you use most, where you get stuck, how long tasks take, what you ask for repeatedly. At the end of a session or week, it generates a reflection report.

The use case is improving your own workflow. If you're asking the same question five times a week, that's a skill you should set up. If you're spending 40% of your time on status updates, that's where automation should go next.

## When to use it

- When you want to understand where your time actually goes
- When you're setting up PM Pilot for the first time and want to know which skills to prioritize
- When you're iterating on your workflow and want data rather than guesses

## How to trigger it

Say: `start meta-observer` at the beginning of a session

Or review logged data: `what patterns did meta-observer find this week`

## What you need

Claude Code CLI. The observer logs to a local file in your memory directory. Nothing leaves your machine. The weekly reflection report is generated on demand and can be added to your weekly-status output.
